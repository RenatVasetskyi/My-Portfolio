# My Portfolio

This repository contains non-commercial Unity mini-projects created by me to demonstrate my coding style, architecture approach, gameplay implementation, UI workflows, and mobile game development experience.

These projects are not commercial releases and are not copies of any company-owned games. They are small portfolio samples built specifically to show how I structure Unity projects, write C# code, organize gameplay systems, work with UI, and approach mobile-oriented development.

## Disclaimer

All source code in this repository was written manually by me.

I did not use AI coding tools, AI code review tools, or AI-generated code for the implementation. The purpose of this repository is to show my own coding style, architecture decisions, problem-solving approach, and how I think when building Unity systems.

AI and stock tools were used only for selected visual `.png` assets, such as concept images or placeholder art. These assets may include materials generated or prepared with tools such as Midjourney, Figma AI, or Freepik.

## Purpose

The goal of this portfolio is to provide technical proof of my Unity experience through small, focused projects that demonstrate:

- Gameplay systems
- Mobile UI flows
- Project architecture
- Clean C# code
- Object pooling
- State machines
- Economy and progression logic
- Performance-conscious mobile development
- Unity workflow and scene organization
